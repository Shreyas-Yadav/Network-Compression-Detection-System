# project-1

## Purpose
This directory contains code for a network compression detection tool. It aims to identify if network traffic is being compressed by sending high and low entropy packet trains and analyzing the timing differences in responses. This helps in understanding network behavior and potential performance impacts due to compression.

## Main Features
*   Reads configuration parameters from a JSON file.
*   Sends TCP SYN packets to initiate connections.
*   Sends high and low entropy UDP packet trains.
*   Listens for TCP RST packets and timestamps them.
*   Calculates checksums for IP and TCP/UDP headers.
*   Detects compression based on timing differences between high and low entropy packet responses.

## Architecture & Design
The code implements a client-server model with a standalone application option. The `compdetect_client.c` and `compdetect_server.c` programs communicate over TCP for configuration and result reporting, and use UDP for the probing phase. The `compdetect.c` program combines both functionalities into a single executable. The system uses raw sockets for packet crafting and analysis, and threads for asynchronous RST packet capture. The design relies on the principle that compressible (low entropy) data will experience different latency than incompressible (high entropy) data if network compression is active.

## Unique / Notable Aspects
*   Uses raw sockets to craft and send custom IP and TCP/UDP packets, providing fine-grained control over packet headers and payloads.
*   Employs a timing-based detection mechanism that infers network compression by analyzing the latency differences between high and low entropy packet trains.
*   Implements a multi-threaded architecture in the standalone application to concurrently listen for RST packets while sending UDP traffic.
